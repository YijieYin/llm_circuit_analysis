# LLM circuit analysis

[Note] Most of the code here is generated by [Claude](https://claude.ai/new) under the instruction of the author(s).

An LLM-powered pipeline for interpreting *Drosophila* neural circuits from [connectome data](https://github.com/YijieYin/connectome_data_prep). Given a set of source and target neuron types, the pipeline automatically infers pairwise pathway mechanisms, per-neuron integration logic, and a circuit-level functional synthesis — all grounded in connectome connectivity and [published literature](https://docs.google.com/spreadsheets/d/1VHCEnurOdb4FDC_NUKZX_BpBckQ9LpKxv0CsK_ObVok/edit?usp=sharing).

Designed to run as SLURM array jobs on a GPU cluster, with support for local inference via [llama.cpp](https://github.com/ggerganov/llama.cpp), OpenAI, or Anthropic.

---

## Modules

```
paper_discovery/                  ← Step 0a (optional): BFS-discover relevant papers, download PDFs
function_extraction_from_papers/  ← Step 0b (optional): extract cell-type functions from PDFs
neuron_interpretation/            ← Step 1: per-neuron functional hypotheses
circuit_analysis/                 ← Step 2: 3-stage circuit-level analysis
llm_core/                         ← Shared LLM utilities (providers, llama-server lifecycle, JSON validation)
```

### Module 0a — Paper discovery
BFS over the Semantic Scholar citation graph starting from seed DOIs/URLs, with a small LLM filtering for relevance. `match_and_download_pdfs.py` then matches discovered papers against existing local PDFs (by DOI then fuzzy title) and downloads missing open-access PDFs from S2 / Unpaywall.

### Module 0b — Function extraction
Reads neuroscience PDFs and uses an LLM to extract structured (cell type → function) mappings. Output `extracted.csv` seeds or extends the `known_types` reference used downstream. Supports `--dump-prompts` for manual use (paste into Claude.ai / ChatGPT) without an LLM API.

### Module 1 — Neuron interpretation
For each neuron type, retrieves its upstream/downstream connectivity (up to `--n-steps` hops, filtered by synaptic-weight threshold) and prompts an LLM to infer a functional hypothesis. Results saved incrementally as JSONL and merged into `hypotheses.csv`. Supports `--dump-prompts`.

### Module 2 — Circuit analysis (3-step pipeline)

| Step | Script | What it does |
|------|--------|--------------|
| 1 | `circuit_step1_pairwise.py` | For every (source → target) pair: finds multi-hop paths, formats them by layer/recipient, asks the LLM to infer the pathway mechanism and net signed effect. Optional actor-critic revision. |
| 2 | `circuit_step2_per_target.py` | For each target: aggregates step-1 analyses and asks how the target integrates its inputs. |
| 3 | `circuit_step3_synthesis.py` | Aggregates step-2 analyses into a circuit-level synthesis: mechanism, adaptive function, predictions, open questions. |

`dump_pair_prompt.py` writes the step-1 prompt for a single (source, target) pair, for manual use without an LLM.

## Setup

```bash
# 1. Conda env + core deps
conda create -n llm python=3.12 && conda activate llm
pip install connectome_interpreter pandas scipy tqdm python-dotenv huggingface_hub \
            openai anthropic rapidfuzz PyMuPDF requests-cache

# 2. Install llm_core (editable, so source edits take effect immediately)
pip install -e .

# 3. API keys (only for providers you intend to use)
cat > .env <<EOF
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
EOF
```

For local inference with llama.cpp on HPC, see [`docs/hpc_llama_setup.md`](docs/hpc_llama_setup.md). No API key needed for `--provider llama`.

`--use-rag` (function extraction only) additionally requires `chromadb`, `tiktoken`, `llama_index`.

## Usage

### Module 0a — paper discovery (optional)
```bash
# Edit seeds.txt with DOIs/URLs, edit paths in paper_discovery_hpc.sh
sbatch paper_discovery/paper_discovery_hpc.sh
sbatch paper_discovery/match_and_download_pdfs.sh
# → discovered_papers_with_local.csv (feeds module 0b's --papers-csv)
```

### Module 0b — function extraction (optional)
```bash
sbatch function_extraction_from_papers/function_extraction_from_paper.sh
python merge_chunks.py --kind extraction \
    --results-dir extraction_results/results \
    --output extraction_results/extracted.csv
```

For manual extraction of a single paper: `--dump-prompts <dir>` writes the system + user prompts to markdown, then paste into Claude.ai / ChatGPT / Gemini and upload the PDF directly.

### Module 1 — neuron interpretation
```bash
sbatch neuron_interpretation/neuron_interpretation.sh
python merge_chunks.py --kind neuron \
    --results-dir results --output hypotheses.csv
```

### Module 2 — circuit analysis
```bash
python circuit_analysis/select_types.py        # → circuit_types.json
bash circuit_analysis/run_pipeline.sh          # submits steps 1→2→3 with SLURM dependencies
# or sbatch run_step{1,2,3}.sh individually
```

For manual analysis of a single pair: `python circuit_analysis/dump_pair_prompt.py --source X --target Y --output prompt.md`.

All long-running scripts support `--resume`: resubmitting a partial run skips already-completed entries.

## LLM providers

Every script accepts `--provider`:

| Provider | Default model | Notes |
|---|---|---|
| `llama` | Qwen3.5-35B-A3B GGUF | Local inference via llama-server; no rate limits |
| `openai` | `gpt-4o` | Also supports `gpt-5`, `gpt-5.4` (Responses API with `--reasoning-effort`, `--verbosity`) |
| `anthropic` | `claude-sonnet-4-6` | Extended thinking via `--thinking-budget` |

Override with `--model <model-name>`.

## Key options

| Module | Flag | Default | Description |
|---|---|---|---|
| circuit | `--side` | `right` | Hemisphere (`left` / `right`) |
| circuit | `--n-steps` | `3` | Max synaptic hops |
| circuit | `--threshold` | `0.01` | Minimum weight to include |
| circuit | `--actor-critic` | off | Critic-revision loop in step 1 |
| circuit | `--shared-intermediates` | off | Convergence analysis in step 2 |
| neuron | `--n-steps` | `4` | Connectivity hops |
| neuron | `--top-n` | `15` | Top up/downstream types shown |
| neuron | `--no-add-function` | — | Skip mapping cell types to known functions |
| extraction | `--mode` | `experimental` | `brief` for 6-word summaries |
| extraction | `--use-rag` | off | ChromaDB RAG instead of full-text input |
| neuron / extraction | `--dump-prompts <dir>` | — | Write prompts to markdown for manual use |

## Parallelism

All long-running scripts accept `--chunk-id` and `--n-chunks` and are designed as SLURM array jobs. Each task processes its chunk independently and writes a separate JSONL file; `merge_chunks.py` combines them after.

## Data dependencies

Paths are set in the SLURM `.sh` wrappers — update them there when data or snapshots change:

| Data | `.sh` files referencing it |
|---|---|
| Connectome (`--base-path`) | `run_step{1,2}.sh`, `neuron_interpretation.sh` |
| Known types CSV (`--known-types-csv`) | as above + `function_extraction_from_paper.sh` |
| Hypotheses CSV (`--hypotheses-csv`) | `run_step{1,2,3}.sh` |
| GGUF model (`--gguf`) | all SLURM wrappers |
| Papers CSV + PDF directory | `function_extraction_from_paper.sh` |
