# llm_circuit_analysis

An LLM-powered pipeline for interpreting *Drosophila* neural circuits from connectome data. Given a set of source and target neuron types, the pipeline automatically infers pairwise pathway mechanisms, per-neuron integration logic, and a circuit-level functional synthesis — all grounded in connectome connectivity and published literature.

Designed to run as SLURM array jobs on a GPU cluster, with support for local inference via [llama.cpp](https://github.com/ggerganov/llama.cpp), OpenAI, or Anthropic.

---

## Overview

The repository contains three semi-independent modules that can be run in sequence:

```
function_extraction_from_papers/   ← Step 0 (optional): mine cell-type functions from PDFs
neuron_interpretation/             ← Step 1: generate per-neuron functional hypotheses
circuit_analysis/                  ← Step 2: 3-stage circuit-level analysis pipeline
docs/                              ← HPC setup guides
```

### Module 1 — Function extraction from papers

Reads a collection of neuroscience PDFs and uses an LLM to extract structured (cell type → function) mappings. The output `extracted.csv` can be used to seed or extend the `known_types` reference used in later stages.

### Module 2 — Neuron interpretation

For each neuron type in a connectome, retrieves its upstream/downstream connectivity (up to `n_steps` hops, filtered by synaptic weight threshold) and prompts an LLM to infer a functional hypothesis. Results are saved incrementally as JSONL and merged into a `hypotheses.csv`.

### Module 3 — Circuit analysis (3-step pipeline)

Takes a user-defined set of source and target neuron types and runs:

| Step | Script | What it does |
|------|--------|--------------|
| 1 | `circuit_step1_pairwise.py` | For every (source → target) pair: finds multi-hop paths, formats them by layer/recipient, and asks the LLM to infer the pathway mechanism and net signed effect. Optional actor-critic revision loop. |
| 2 | `circuit_step2_per_target.py` | For each target: aggregates all step-1 pairwise analyses and asks how the target integrates its inputs. Optionally identifies shared intermediates (neurons in paths from 2+ sources). |
| 3 | `circuit_step3_synthesis.py` | Reads all step-2 analyses and produces a single circuit-level synthesis: overall computation, adaptive function, testable predictions, and open questions. |

## Setup

### 1. Python environment

```bash
conda create -n llm python=3.12
conda activate llm
pip install connectome_interpreter pandas scipy tqdm python-dotenv huggingface_hub
```

### 2. API keys

Create a `.env` file in the repo root:

```env
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
```

Keys are only required for the provider(s) you intend to use. For local inference with llama.cpp, no API key is needed.

### 3. Local inference with llama.cpp (recommended for HPC)

See [`docs/hpc_llama_setup.md`](docs/hpc_llama_setup.md) for a step-by-step guide to building llama.cpp with CUDA and downloading a GGUF model (tested with Qwen3.5-35B-A3B on A100).

## Usage

### Module 1 — Extract functions from papers

First, run `paper_getting.ipynb` locally to generate `papers_with_names.csv`, then copy it and your PDFs to the HPC (see [`docs/get_paper_to_hpc.md`](docs/get_paper_to_hpc.md)).

Edit paths in `function_extraction_from_paper.sh`, then submit:

```bash
cd function_extraction_from_papers
sbatch function_extraction_from_paper.sh

# After all chunks complete, merge:
python merge_extraction_results.py --results-dir extraction_results/results --output extraction_results/extracted.csv
```

The merged CSV can then be normalised using `normalize_cell_types.py`.

---

### Module 2 — Neuron interpretation

Edit paths in `neuron_interpretation.sh`, then submit:

```bash
cd neuron_interpretation
sbatch neuron_interpretation.sh

# After all chunks complete, merge:
python neuron_interpretation_merge_results.py \
    --results-dir results \
    --output hypotheses.csv
```

The output `hypotheses.csv` is used as input for the circuit analysis pipeline.

---

### Module 3 — Circuit analysis

#### Step 0: Define your circuit

Edit `select_types.py` to filter the cell types you want to analyse (by known function, super class, or explicit list), then run:

```bash
cd circuit_analysis
python select_types.py
# → produces circuit_types.json
```

#### Option A: Submit the full pipeline in one go

```bash
bash run_pipeline.sh           # submits steps 1→2→3 with SLURM dependencies
bash run_pipeline.sh --dry-run # preview commands without submitting
```

#### Option B: Submit steps individually

```bash
sbatch run_step1.sh            # pairwise pathway analysis (array job)
sbatch run_step2.sh            # per-target integration (array job, after step 1)
sbatch run_step3.sh            # circuit synthesis (single job, after step 2)
```

All scripts support `--resume`: resubmitting a failed or partial run will skip already-completed entries.

#### Output files

| Path | Contents |
|------|----------|
| `results_step1/pairwise_chunk_*.jsonl` | One record per (source, target) pair with pathway mechanism, net signed effect, and optional critic flags |
| `results_step2/target_chunk_*.jsonl` | One record per target with integration mechanism and refined hypothesis |
| `results_step3/circuit_synthesis.json` | Full circuit synthesis: mechanism, adaptive function, predictions, open questions |

## LLM providers

All scripts accept a `--provider` flag:

| `--provider` | Default model | Notes |
|---|---|---|
| `llama` | Qwen3.5-35B-A3B (GGUF) | Local inference via llama-server; no rate limits or API costs |
| `openai` | `gpt-4o` | Also supports `gpt-5`, `gpt-5.4` (Responses API with `--reasoning-effort` and `--verbosity`) |
| `anthropic` | `claude-sonnet-4-6` | Extended thinking via `--thinking-budget` |

Override the default model with `--model <model-name>`.

## Key options

### Circuit analysis (steps 1–3)

| Flag | Default | Description |
|------|---------|-------------|
| `--side` | `right` | Hemisphere to analyse (`left` or `right`) |
| `--n-steps` | `3` | Max synaptic hops for path finding |
| `--threshold` | `0.01` | Minimum synaptic weight to include |
| `--max-tokens` | `20000` | Max tokens per LLM response |
| `--resume` | on | Skip already-processed entries on resubmission |
| `--actor-critic` | on | Enable critic-revision loop in step 1 |
| `--max-critic-rounds` | `1` | Max revision rounds per pair |
| `--shared-intermediates` | off | Include convergence analysis in step 2 |

### Neuron interpretation

| Flag | Default | Description |
|------|---------|-------------|
| `--n-steps` | `4` | Connectivity hops |
| `--top-n` | `15` | Top upstream/downstream types to include |
| `--no-add-function` | off | Skip mapping cell types to known functions |

### Function extraction

| Flag | Default | Description |
|------|---------|-------------|
| `--mode` | `experimental` | `brief` for 6-word summaries, `experimental` for structured extraction |
| `--max-chars` | — | Truncate very long PDFs |
| `--think` | off | Enable Qwen3 chain-of-thought reasoning |
| `--use-rag` | off | Use ChromaDB RAG instead of full-text input |

## Parallelism and chunking

All long-running scripts accept `--chunk-id` and `--n-chunks` and are designed to be run as SLURM array jobs. Each job independently processes its chunk of the input (neuron types or source-target pairs) and writes to a separate JSONL file. Results are merged after all chunks complete.

The number of SLURM array tasks (`--array=0-N`) must match `N_CHUNKS` set in the shell script. For step 1 the natural chunk size is over source-target pairs; for step 2 it is over target types.

## Data dependencies

The pipeline expects the following external data (not included in this repo). The paths to each are set in the SLURM shell scripts — update them there when data or snapshots change:

| Data | Where to update |
|------|-----------------|
| Connectome data (`--base-path`) | `BASE_PATH` in `run_step1.sh`, `run_step2.sh`, `neuron_interpretation.sh` |
| Known types CSV (`--known-types-csv`) | `KNOWN_TYPES_CSV` in `run_step1.sh`, `run_step2.sh`, `neuron_interpretation.sh`, `function_extraction_from_paper.sh` |
| Neuron hypotheses CSV (`--hypotheses-csv`) | `HYPOTHESES_CSV` in `run_step1.sh`, `run_step2.sh`, `run_step3.sh` |
| GGUF model (`--gguf`) | `GGUF` in `run_step1.sh`, `run_step2.sh`, `run_step3.sh`, `neuron_interpretation.sh`, `function_extraction_from_paper.sh` |
| Papers CSV and PDF directory | `PAPERS_CSV` and `PAPERS_DIR` in `function_extraction_from_paper.sh` |
